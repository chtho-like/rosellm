# cuda/flashattn

Goal: on the current machine (RTX 4070 / SM89), build **scaled dot-product
attention (SDPA) forward** as a step-by-step optimization ladder, then use a
unified benchmark script and plots to compare every step with the production-grade
FlashAttention implementation (`flash-attn`).

Notes:

- This directory focuses on **Triton kernels**, which make rapid experimentation,
  autotuning, and Tensor Core code generation easier.
- The benchmark also includes explicitly selected PyTorch SDPA backends:
  - `torch_sdpa_flash_pytorch`: PyTorch built-in Flash-Attention backend
  - `torch_sdpa_efficient_cutlass`: PyTorch memory-efficient attention (CUTLASS)
  - `torch_sdpa_cudnn`: cuDNN SDPA backend
- Every optimization version has its own file: `triton_flashattn_*.py`.
- `bench_flashattn.py` provides the unified benchmark, persists results, and
  generates plots.

## Requirements

- An NVIDIA GPU (the local GPU is an RTX 4070 / SM89)
- `python`, `torch`, and `triton` (already installed locally)
- Optional: install the production-grade `flash-attn` package as a comparison baseline

## Quick start

Run the benchmark directly from this directory:

```bash
cd /data/projects/rosellm/notebooks/cuda/flashattn
python bench_flashattn.py \
  --causal 1 --dtype fp16 \
  --batch 1 --heads 32 \
  --seq-lens 256,512,1024,2048,4096 \
  --head-dims 64,128 --check
```

To include the production-grade `flash-attn` implementation
(Dao-AILab/flash-attention), install it as follows to avoid compiling unused
GPU architectures:

```bash
FLASH_ATTN_CUDA_ARCHS=80 NVCC_THREADS=8 \
  python -m pip install --no-cache-dir flash-attn==2.7.3
```

Outputs:

- `./out/latest/results.json`
- `./out/latest/results.csv`
- `./out/latest/plots/*.png`

To delegate Triton shape-to-tile configuration selection to autotuning as much
as possible, first run offline tuning to generate the v9 configuration table:

```bash
python tune_triton_flashattn_table.py \
  --device 0 --causal 1 --dtype fp16 \
  --batch 1 --heads 32 \
  --seq-lens 256,512,1024,2048,4096 \
  --head-dims 64,128
```

This generates `triton_flashattn_v9_table_sm89.json` (named after the SM version
by default). You can override its path with an environment variable:

```bash
FLASHATTN_TRITON_AUTOTUNE_TABLE=/path/to/table.json \
  python bench_flashattn.py ...
```

## File progression (easy to advanced)

- `triton_flashattn_0_baseline.py`: minimal working FlashAttention (online softmax)
- `triton_flashattn_1_codegen_hints.py`: adds code-generation hints such as
  `tl.multiple_of` and alignment information
- `triton_flashattn_2_autotune.py`: adds autotuning over blocks, warps, and stages
- `triton_flashattn_3_fixed_best.py`: fixes the best configuration measured locally
- `triton_flashattn_4_causal_cutoff.py`: eliminates invalid K-block work for causal attention
- `triton_flashattn_5_even_fastpath.py`: mask-free fast path for evenly divisible dimensions
- `triton_flashattn_6_autotune_cutoff.py`: autotunes the cutoff and fast-path implementation
- `triton_flashattn_7_fixed_best_cutoff.py`: fixes the strongest local configuration
  (the best at this stage)
- `triton_flashattn_8_dispatch_fixed_best.py`: dispatches fixed best configurations
  by `head_dim` and `seq_len`, making it more suitable for direct use
- `triton_flashattn_9_autotune_table.py`: dispatches through a shape-to-configuration
  table generated by offline autotuning; this approximates automatic tile selection
  while avoiding runtime autotuning
